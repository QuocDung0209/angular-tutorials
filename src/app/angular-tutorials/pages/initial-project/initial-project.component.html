Tạo project với Angular CLI (v12)

Chỉ định packages manager - yarn cho project

ng config -g cli.packageManager yarn

Tạo project với tên project-name

ng new project-name --style=scss --skip-git --routing
Sau khi project được tạo, hãy tạo 2 modules: CoreModule và SharedModule theo Best practices folder structure.

Khởi tạo Git để quản lý project

  1. git init
  2. git remote add origin remote-url

Install and config Prettier

Cài đặt các packages cần thiết

yarn add --dev prettier eslint-config-prettier eslint-plugin-prettier

Tạo file .prettierrc.json và paste code bên dưới vào

Install and config ESLint

Thêm package @angular-eslint/schematics

ng add @angular-eslint/schematics

Thay toàn bộ code bên trong file .eslintrc.json bằng code bên dưới

Thêm script để tự động sửa các lỗi và warning từ ESLint và Prettier

Update packages.json file

Sau khi cấu hình xong, hãy chạy lệnh yarn lint:fix để tự động fix các errors và warning từ ESLint và Prettier.

Cài đặt sort-import extension cho VSCode để sắp xếp các import theo thứ tự bạn chữ cái.

Install Bootstrap

  1. yarn add bootstrap
  2. Add "./node_modules/bootstrap/dist/css/bootstrap.min.css" to architect -> build -> styles of angular.json

Install Fontawesome

  1. yarn add font-awesome
  2. Add "./node_modules/font-awesome/css/font-awesome.css" to architect -> build -> styles of angular.json

Các Extension cho Visual Studio Code hữu ích khi code React

  • Atom One Dark Theme: Theme Atom cho VS Code, đẹp hơn theme mặc định nhiều.
  • Auto Close Tag: Tự động đóng tag khi code HTML, JSX
  • Auto Rename Tag: Tự động đổi tên tag khi code HTML, JSX. Kết hợp với Auto Close Tag sẽ thành bộ đôi tuyệt vời
  • Bookmarks: Đánh dấu dòng code để tìm kiếm dễ dàng. Khi code bạn dài ngoằn, việc phải scroll lên, scroll xuống để tìm và code khá mệt mỏi. Lúc này chỉ cần đánh dấu vị trí, sau đó bạn chỉ cần trỏ đến vị trí đó là editor sẽ mở ngay tại vị trí bookmark luôn.
  • Bracket Pair Colorizer: Giúp phân biệt các dấu {()} khi nó trở nên quá nhiều
  • Code Spelling Checker: Kiểm tra lỗi chính tả khi code (tiếng Anh thôi nhé)
  • EditorConfig: Để chạy được file .editorconfig.
  • ESLint: ESLint cho editor
  • GitLens: Quản lý code tốt hơn với git, hiện thông tin người thay đổi file, commit,…
  • html to JSX: Chuyển HTML thành JSX
  • Live Server: Tạo server auto reload khi code html tĩnh
  • Material Icon Theme: Thêm bộ theme icon cho các file, folder giúp bạn nhận biết file nhanh chóng
  • Prettier: Format code đẹp, xịn
  • SVG Viewer: Xem ảnh svg trên editor
  • vscode-styled-components: Highlight và auto-complete cho styled-component như khi code trên file css vậy
  • Angular Language Service: Gợi ý các function, directive, component Angular khi viết template,...

Cài đặt Workspace setting cho VS Code

VS Code có 2 dạng setting, một là cho user, hai là cho workspace. Nếu chúng ta không set workspace thì mặc định VS Code sẽ lấy setting ở user. Và setting cũng có 2 cách setting, 1 là setting bằng file JSON, 2 là bằng UI.

Setting VS Code workspace và .editorconfig

  1. Tạo file ./vscode/settings.json ở thư mục chính (root folder):

    Giải thích:

    • Chọn Default Formatter là Prettier
    • "editor.formatOnSave": true để khi lưu thì sẽ tự động format (bằng ESLint hay bằng Prettier thì như ta đã định nghĩa phía trên)
    • "eslint.format.enable": true để bật tính năng format code của ESLint.
    • "source.fixAll.eslint": true để khi lưu thì ESLint sẽ tự động format và fix lỗi ESLint cho chúng ta luôn.
  2. Bổ sung thêm recommended extension vào workspace, để khi ai mở source code lên thì VS Code sẽ khuyên cài thêm các extension này. Chúng ta thêm file extensions.json trong folder .vscode nhé. Đây là các extension đã liệt kê phía trên, chúng ta chỉ cần copy file này bỏ vào source thì editor sẽ hiện lên các recommended extension, không cần tìm kiếm từng cái.

    Điều này vô cùng hữu ích khi làm việc nhóm, để thống nhất các extension và setting giữa các member với nhau thì ta nên share cái folder .vscode (cứ để nguyên nó trong source, đừng bỏ vào .gitignore là được 😛 ).

  3. Thêm file .editorconfig trong root folder - Cần cài Editor Config extension cho VS Code

    EditorConfig sinh ra để thống nhất các chuẩn giữa các editor khác nhau. Ví dụ người thì dùng indent là space, người thì dùng app-tab; người thì indent là 2, người thì là 4. EditorConfig giúp code chúng ta có tính thống nhất, dễ đọc và dễ bảo trì hơn.
